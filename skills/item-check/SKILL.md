# item-check — 린캔버스로 "검증 착수 준비" 판정

## 이 스킬이 존재하는 이유 (실제 시행착오)
> 한 팀이 린캔버스의 **수요 칸을 '가상 사례'로 채워놓고** "다 채웠으니 검증됐다"고 착각한 채
> 랜딩페이지를 12개 만들었다. 2개월이 날아갔다.
> → **칸을 다 채움 = "검증 완료"가 아니다. "랜딩으로 검증 착수할 준비 됨"일 뿐이다.**

## 방법론: Lean Canvas (Ash Maurya)
정해진 9칸을 채운다. **빈 칸 = 부족한 점**(아직 착수 이르다). 다 채우면 랜딩으로 검증 착수.
Mom Test·RAT는 별도 프로세스가 아니라 **칸을 채우는 도구**로만 쓴다(아래).

## 안전핀 하나 — 수요 칸은 근거 태그
수요에 직결된 4칸(**Problem · Customer · UVP · Unfair Advantage**)만 **[근거 있음 / 아직 가정]** 을 붙인다.
- **근거** = 관찰된 과거 행동·실제 인입·인터뷰·계약 (Mom Test 기준: 의견·칭찬·"쓸 것 같다"는 근거 아님)
- **가정** = 머릿속 예시·추측
- 다 채웠어도 수요 칸이 죄다 '가정'이면 → "채웠지만 미검증. **랜딩으로 바로 이걸 확인하는 것**" 경고.

## 9칸과 채우기 가이드
| # | 칸 | 채울 내용 | 근거태그 |
|---|---|---|---|
| ① | **Problem** (+ 기존 대안) | 상위 1~3개 문제 + 지금 사람들이 그걸 어떻게 때우는지 | ✔ |
| ② | **Customer Segments** (+ 얼리어답터) | 누구인가 + 가장 먼저 살 소수는 누구 | ✔ |
| ③ | **UVP** (+ 하이레벨 컨셉) | 왜 다른가 한 문장 + "X의 Y" 식 비유 | ✔ |
| ④ | **Solution** | 각 문제에 대응하는 최소 기능 | |
| ⑤ | **Channels** | 얼리어답터에 닿는 경로 | |
| ⑥ | **Revenue** | 과금 방식·가격 | |
| ⑦ | **Cost** | 주요 비용 | |
| ⑧ | **Key Metrics** | 검증할 핵심 숫자(랜딩이 잡을 신호) | |
| ⑨ | **Unfair Advantage** | 쉽게 복제 못 하는 것 (없으면 빈칸으로 두고 위험 표시) | ✔ |

## 순서
1. 아이템을 한두 문장으로 받는다. 모호하면 되묻는다.
2. 9칸을 채운다. 못 채우는 칸은 비워두고 **왜 못 채우는지 + 어떻게 채울지** 코칭:
   - "어느 빈칸부터?" → **RAT**: 틀리면 죽는 가장 위험한 칸부터.
   - "수요 칸 근거를 어떻게?" → **Mom Test**: "쓰실래요?"(금지) 대신 "최근에 이 문제로 뭘 하셨어요?"(과거 행동).
3. 수요 4칸에 근거 태그.
4. **판정**:
   - ✅ **착수 준비 완료** — 9칸 다 참 + 수요 칸에 근거 최소 1개
   - ◐ **조건부** — 핵심 칸(①②③)은 찼으나 일부 빈칸/가정 → 보완 후 착수
   - ❌ **아직 이르다** — 핵심 칸이 비었거나 전부 가정 → 랜딩 만들지 말고 먼저 채워라
5. `lean-canvas.md`로 저장.

## 채운 캔버스 → 랜딩 매핑 (이게 다음 단계로 이어짐)
| 캔버스 칸 | 랜딩 요소 |
|---|---|
| UVP + 하이레벨 컨셉 | Hero 헤드라인 = **키카피** |
| Problem + 기존 대안 | 문제 제기 섹션 |
| Solution | 작동 방식 / 쇼케이스 |
| Customer + 얼리어답터 | 타겟·톤 |
| Revenue | 가격·오퍼·CTA |
| Unfair Advantage | 차별점 배지 |
| Key Metrics | CTA가 잡을 검증 신호 |
| Channels / Cost | 내부용(랜딩 미노출) |
