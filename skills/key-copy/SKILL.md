# key-copy — 테스트할 키카피 N개

키카피 = 아이템의 **핵심 한 줄**(헤드라인/가치 제안). 이 스킬은 그걸 **한 개로 찍지 않고 N개 변형**으로 뽑아,
어떤 앵글이 시장에 먹히는지 테스트하게 한다.

## 전제
- 가능하면 `item-check.md`를 먼저 읽는다. 리스키스트 가정·타겟·지불의사가 카피 앵글을 정한다.
- item-check 판정이 ❌부적절이면 **카피를 뽑지 않는다.** 아이템부터 고친다.

## N 정하기
- 사용자가 개수를 주면 그대로. 안 주면 **기본 5개.**
- 각 카피는 **서로 다른 소구 앵글**을 하나씩 담당한다(같은 말 바꿔쓰기 금지).

## 소구 앵글 팔레트 (서로 다른 것끼리 고른다)
- **문제/고통** — 지금 겪는 아픔을 찌른다
- **성과/결과** — 얻게 될 구체적 결과(숫자)
- **타이밍** — "지금" 필요한 이유
- **정체성** — "이런 사람을 위한 것"
- **가격/절감** — 앵커링(기존 비용 대비)
- **반전/네거티브** — 통념을 뒤집기

## 순서
1. `item-check.md`(있으면)와 아이템을 읽는다.
2. 앵글을 N개 고른다(중복 앵글 금지).
3. 앵글마다 키카피 1개씩 작성. **실제 문구 크래프트는 companion 스킬에 위임**(아래).
4. 각 카피에 **[테스트하는 앵글] + [왜 이 표현인지] 근거**를 붙인다.
5. 검증 못 한 수치는 `[플레이스홀더]`로 표기(있는 척 금지).
6. `key-copy.md`로 저장.

## 산출물 템플릿 (`key-copy.md`)
```
## 아이템: (한 문장)  |  N = (개수)

| # | 키카피            | 앵글      | 왜(근거)                    |
|---|-------------------|-----------|-----------------------------|
| 1 | "…"               | 문제/고통 |                             |
| 2 | "…"               | 성과      |                             |
| 3 | "…"               | 타이밍    |                             |
| … |                   |           |                             |

추천: 먼저 테스트할 2~3개 = (앵글이 가장 갈리는 것들)
```

## Companion 스킬 (있으면 재사용, 없어도 동작)
카피 크래프트는 이미 만들어둔 범용 스킬이 담당한다. 설치돼 있으면 호출해 쓴다(중복 구현 금지):
- **`premium-marketing`** — 미저리·미라클, 상·욕·방·보, 앵커링·넛지. ← 문제/성과/가격 앵글에 직접 활용
- **`brand-story`** — 정체성/서사 앵글이 필요할 때.

> 두 스킬은 독립 스킬이라 이 플러그인에 없다. 없어도 key-copy 자체로 변형은 나오지만,
> 전환·서사 품질을 높이려면 설치(README 참조)해 함께 쓰길 권한다.
