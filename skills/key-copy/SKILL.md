# key-copy — 캔버스에서 뽑는 키카피 N개

키카피 = 랜딩의 **핵심 한 줄(Hero 헤드라인)**. 별도 창작이 아니라 **린캔버스의 UVP·Problem 칸에서 파생**한다.
한 개로 찍지 않고 **앵글이 다른 N개 변형**으로 뽑아, 무엇이 먹히는지 테스트하게 한다.

## 전제
- `lean-canvas.md`를 먼저 읽는다. 없으면 `/item-check` 먼저.
- item-check 판정이 ❌(아직 이르다)면 **카피를 뽑지 않는다.** 캔버스 빈칸부터 채운다.
- 소스 칸: **UVP + 하이레벨 컨셉**(핵심 메시지), **Problem + 기존 대안**(앵커링), **Customer**(톤).

## N 정하기
- 사용자가 개수를 주면 그대로. 안 주면 **기본 5개.**
- 각 카피는 **서로 다른 소구 앵글**을 하나씩 담당(같은 말 바꿔쓰기 금지).

## 소구 앵글 팔레트 (서로 다른 것끼리)
- **문제/고통** — Problem 칸을 찌른다
- **성과/결과** — UVP를 구체적 결과(숫자)로
- **타이밍** — "지금" 필요한 이유
- **정체성** — Customer 칸 "이런 사람을 위한 것"
- **가격/절감** — 기존 대안 비용 대비 앵커링
- **반전/네거티브** — 통념 뒤집기

## 순서
1. `lean-canvas.md`와 아이템을 읽는다.
2. 앵글 N개 고른다(중복 금지). UVP가 핵심 메시지, Problem이 고통, 기존 대안이 앵커.
3. 앵글마다 키카피 1개. **실제 문구 크래프트는 companion 스킬에 위임**(아래).
4. 각 카피에 **[앵글] + [어느 칸에서 나왔는지 + 왜]** 근거를 붙인다.
5. 검증 안 된 수치는 `[플레이스홀더]`로 표기(있는 척 금지).
6. `key-copy.md`로 저장.

## 산출물 템플릿 (`key-copy.md`)
```
## 아이템: (한 문장)  |  N = (개수)

| # | 키카피 | 앵글 | 출처 칸 / 왜 |
|---|--------|------|--------------|
| 1 | "…"    | 문제 | Problem / … |
| 2 | "…"    | 성과 | UVP / …     |
| … |        |      |             |

추천 우선 테스트 2~3개: (앵글이 가장 갈리는 것들)
```

## Companion 스킬 (있으면 재사용, 없어도 동작)
- **`premium-marketing`** — 미저리·미라클, 상·욕·방·보, 앵커링·넛지. ← 문제/성과/가격 앵글에 활용
- **`brand-story`** — 정체성/서사 앵글이 필요할 때

> 두 스킬은 독립 스킬이라 이 플러그인에 없다. 없어도 변형은 나오지만, 품질을 높이려면 설치(README) 권장.
