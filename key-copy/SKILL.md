# key-copy — 캔버스 × 스타일 = 키카피 N개

## 규율 (겉으로 설명하지 말고 그냥 적용)
- 카피는 캔버스 **UVP·Problem 칸**에서. 한 개로 안 찍고 **N개 변형**.
- 다양성 축 = **카피 스타일**(톤·접근). ⚠️ 라벨에 **회사명 금지**(토스식·배민식·Linear식 X) → 스타일 속성으로(구어체·안심형, 손실 강조형 등). 레퍼런스 사이트를 줘도 톤 참고만, 라벨은 스타일로.
- 방법론 용어 노출 X. **결과(표)만.**

## 전제 (어느 아이디어인지 먼저)
- `ideas/<slug>/lean-canvas.md` 를 읽는다. slug는 인자로 주면 그걸, 없으면 **가장 최근 수정된 `ideas/*/lean-canvas.md`**. `ideas/`가 여러 개고 애매하면 목록 보여주고 물어본다.
- `ideas/`는 **사용자 작업 폴더(cwd)** 기준. 스킬 폴더(`.claude/skills`)에 쓰거나 읽지 않는다.
- 없으면 `/item-check` 먼저.
- 판정 **❌면 기본은 안 뽑는다.** 단 ❌ 사유가 "수요 미검증"이고 사용자가 **가짜문(대기신청 측정)** 목적이면, "수요 측정용(실서비스 홍보 아님)"임을 한 줄 명시하고 뽑는다.

## N & 스타일
- 개수 안 주면 **5개**. 각기 다른 스타일(중복 금지).
- 스타일 팔레트(라벨 예): **구어체·안심형 / 공감형·문제 직격 / 간결·단정 스펙형 / 손실 강조형(수치) / 정체성 선언형 / 반전형**.
- **레퍼런스 사이트를 주면** → 그 사이트의 톤(문장 길이·구어체/단정체·감정 vs 스펙)을 **참고**해 스타일을 정한다. 단 **출력 라벨은 회사명이 아니라 스타일로**(예: 사이트가 토스여도 라벨은 "구어체·안심형").

## 출력 — 표 하나 + 한 줄
```
| # | 키카피 | 스타일 | 출처칸 |
| 1 | "…"    | 구어체·안심형    | UVP |
| 2 | "…"    | 공감형·문제 직격 | Problem |
```
+ "먼저 테스트 2~3개: (스타일이 가장 갈리는 것)" 한 줄.
- 검증 안 된 수치는 `[플레이스홀더]`.
- `ideas/<slug>/key-copy.md` 로 저장(캔버스와 같은 폴더, cwd 기준. 스킬 폴더에 쓰지 말 것).

## 다음 한 줄: `/landing <slug>` (캔버스+카피 → 랜딩). 캔버스 바뀌면 다시 돌려 최신화.

## Companion (있으면 재사용, 없어도 동작 — 조용히)
`premium-marketing`(앵커링·넛지·상욕방보) / `brand-story`(정체성·서사). 없어도 변형은 나온다.
