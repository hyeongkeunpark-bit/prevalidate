# item-check — 린캔버스 표로 "검증 착수 준비" 판정

## 이 스킬이 존재하는 이유 (실제 시행착오)
> 한 팀이 린캔버스의 **수요 칸을 '가상 사례'로 채워놓고** "다 채웠으니 검증됐다"고 착각한 채
> 랜딩페이지를 12개 만들었다. 2개월이 날아갔다.
> → **칸을 다 채움 = "검증 완료"가 아니라 "랜딩으로 검증 착수할 준비 됨"일 뿐이다.**

## 결과물 = 살아있는 린캔버스 표
매 턴 **표**를 출력한다. 채운 칸 / 못 채운 칸이 한눈에 보이고, 못 채운 칸은 계속 채우도록 돕는다.
사용자가 정보를 가져올수록 표가 갱신되고, 표가 갱신되면 `key-copy`도 다시 뽑는다(한 루프).

## 칸은 두 종류 — 누가 채우나
- 🤖 **내가 초안 낼 수 있는 칸** — 대화 내용으로 초안 작성(단 수요 관련은 근거 태그 필수)
- 🙋 **당신이 실데이터로 채울 칸** — 내가 그럴듯하게 지어내면 안 되는 칸. **짧은 리서치 힌트**만 준다.
  경쟁사·차별점처럼 니치 사실은 내가 모른다. 지어내는 순간 그게 "가정을 사실로 착각"하는 함정.

## 9칸
| # | 칸 | 누가 | 채울 내용 | 근거태그 |
|---|---|---|---|---|
| ① | **Problem** | 🤖 | 상위 1~3개 문제 | ✔ |
| ①b | **기존 대안**(경쟁사) | 🙋 | 지금 사람들이 어떻게 때우나 / 경쟁 제품 | ✔ |
| ② | **Customer** (+얼리어답터) | 🤖초안/🙋근거 | 누구 + 가장 먼저 살 소수 | ✔ |
| ③ | **UVP** (+하이레벨 컨셉) | 🤖 | 왜 다른가 한 문장 + "X의 Y" 비유 | ✔ |
| ④ | **Solution** | 🤖 | 문제별 최소 기능 | |
| ⑤ | **Channels** | 🤖후보/🙋검증 | 얼리어답터에 닿는 경로 | |
| ⑥ | **Revenue** | 🤖방식/🙋실가격 | 과금 방식·가격 | |
| ⑦ | **Cost** | 🙋 | 주요 비용 | |
| ⑧ | **Key Metrics** | 🤖제안 | 검증할 핵심 숫자(랜딩이 잡을 신호) | |
| ⑨ | **Unfair Advantage** | 🙋 | 쉽게 복제 못 하는 것 | ✔ |

## 안전핀 — 수요 칸(✔) 근거 태그
✔ 칸(Problem·기존대안·Customer·UVP·Unfair Advantage)은 **[근거 있음 / 아직 가정]** 을 붙인다.
- **근거** = 관찰된 과거 행동·실제 인입·인터뷰·계약 (Mom Test: 의견·칭찬·"쓸 것 같다"는 근거 아님)
- 다 채웠어도 ✔ 칸이 죄다 '가정'이면 → "채웠지만 미검증. **랜딩으로 바로 이걸 확인하는 것**" 경고.

## 🙋 칸 리서치 힌트 (경쟁/차별 — 지어내지 말 것)
못 채우면 아래를 **짧게** 던진다(리서치 강의로 만들지 말 것):
- 경쟁사 2~3곳을 정하고 각각 확인: **①가격·최소 조건 ②셀프서브/문의경로 ③공개 성과사례 유무 ④최근 불만**(리뷰·커뮤니티)
- 그중 "남들은 못 하는데 우리는 하는 것" 하나 → **Unfair Advantage**
- 못 찾으면 UA는 **빈칸으로 두고 '차별점 미확보' 위험 표시** (가짜로 채우지 말 것)

## 순서
1. 아이템을 한두 문장으로 받는다. 모호하면 되묻는다.
2. 🤖 칸은 초안 작성, 🙋 칸은 리서치 힌트. 못 채운 칸을 표에 명시.
3. "어느 빈칸부터?" → **RAT**: 틀리면 죽는 가장 위험한 칸부터. "수요 근거 어떻게?" → **Mom Test** 과거행동 질문.
4. ✔ 칸 근거 태그.
5. **판정**: ✅ 착수 준비 완료(9칸 참+수요 근거 최소 1개) / ◐ 조건부(①②③ 참, 일부 빈칸·가정) / ❌ 아직 이르다(핵심 칸 빔·전부 가정).
6. `lean-canvas.md`로 저장(표 형태).

## 채운 캔버스 → 랜딩 매핑
| 캔버스 칸 | 랜딩 요소 |
|---|---|
| UVP + 하이레벨 컨셉 | Hero 헤드라인 = **키카피** |
| Problem + 기존 대안 | 문제 제기 섹션 |
| Solution | 작동 방식 / 쇼케이스 |
| Customer + 얼리어답터 | 타겟·톤 |
| Revenue | 가격·오퍼·CTA |
| Unfair Advantage | 차별점 배지 |
| Key Metrics | CTA가 잡을 검증 신호 |
