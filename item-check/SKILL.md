# item-check — 린캔버스로 "검증 착수 준비" 판정

## 규율 (겉으로 설명하지 말고 그냥 적용)
- 만들기 전 점검. **칸 다 채움 = "검증 완료"가 아니라 "랜딩으로 착수 준비 됨"**.
- 수요 칸을 가정으로 채우고 "됐다"고 착각하는 게 최대 함정 → **가정은 가정이라고 표시**.
- 내가 모르는 니치 사실(경쟁사·차별점)은 **지어내지 말고** `확인필요`로.
- 방법론(Lean Canvas·Mom Test·RAT)은 근거로만. **출력에 용어를 드러내지 않는다. 결과만.**

## 출력 형식 — 딱 3블록 + 다음 한 줄 (장황 금지)

**1) 컴팩트 린캔버스 표** (한 줄/칸)
`| 칸 | 내용(짧게) | 상태 |`
- 상태 열은 **수요 칸만**: `가정` / `근거` / `확인필요`. 나머지 칸은 채우면 비움, 비었으면 `⚠️빈칸`.
- 수요 칸 = Problem · 기존대안 · Customer · UVP · Unfair Advantage.
- 내가 모를 칸(기존대안·Unfair Advantage)은 칸 이름 뒤에 🙋 + `확인필요`.
- Channels·Cost·Key Metrics는 짧게, 지면 아끼면 한 줄로 묶어도 됨.

**2) 판정 한 줄**
✅ 착수 준비 / ◐ 조건부(무엇 보완) / ❌ 아직 이르다(무엇 먼저) — 이유 반 줄.

**3) ⚠️ 지금 딱 하나**
- 틀리면 죽는 **가정 1개**를 한 문장으로.
- 그걸 확인할 **질문 3~4개** (의견·"쓸래요?" 금지 → 과거 행동만).
- "몇 명 중 몇이면 진행 / 아니면 재정의" 한 줄.
- ※ RAT·Mom Test라는 말은 쓰지 않는다 — 그냥 물어볼 것만 준다.

**다음 한 줄**: `/key-copy` 로 그 가정 찌르는 카피 (또는 🙋칸 채우고 재실행).

## 순서
1. 아이템 한두 문장 받기(모호하면 되묻기).
2. 9칸 채우기 — 내가 초안 낼 건 초안, 경쟁·차별점은 `확인필요`로 비워두고 지어내지 않음.
3. 수요 칸 가정/근거 판정.
4. 위 3블록으로 출력. **같은 컴팩트 표**를 `lean-canvas.md`로 저장.
5. 판정 ❌면 카피·랜딩으로 가지 말 것.

## 캔버스 → 랜딩 매핑 (참고용, 출력엔 넣지 않음)
UVP=Hero 헤드라인=키카피 / Problem=문제섹션 / Solution=기능 / 기존대안=경쟁비교 / Customer=타겟·얼리 / Revenue=가격·CTA / Key Metrics=CTA 신호.
