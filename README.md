# prevalidate

> **만들기 전에 아이템부터 점검**하는 Claude Code 스킬 묶음.
> 검증하려는 사업/서비스 아이템이 **적절한지 판정**하고, 테스트할 **키카피 N개**를 뽑아준다.

## 왜 만들었나
한 팀이 아이템의 수요를 "있는 것"으로 가정한 채 랜딩페이지를 **12개** 만들었다.
그 "수요 사례"는 나중에 **가상 예시**로 판명됐고, 2개월이 날아갔다.
→ 만들기 전에 **아이템 자체가 검증할 가치·검증 가능성이 있는지**부터 냉정하게 본다.

## 스킬 2개

| 순서 | 스킬 | 하는 일 |
|---|---|---|
| ① | **item-check** | 아이템이 적절한지 6기준으로 판정(문제 실재성·수요 증거·차별성·도달성·지불의사·검증 가능성) + 리스키스트 가정 + 적절/조건부/부적절 판정 |
| ② | **key-copy** | 서로 다른 소구 앵글로 키카피 **N개**(기본 5) 추출. 각 카피가 어떤 앵글을 테스트하는지 명시 |

흐름:
```
아이템
 └─ /item-check → item-check.md  (적절? 리스키스트 가정? 가장 싼 검증법?)
      └─ (적절/조건부일 때만) /key-copy → key-copy.md  (테스트할 키카피 N개)
```

## Companion 스킬 (독립 스킬 — 별도 설치, 있으면 자동 재사용)
카피 크래프트는 검증된 범용 스킬에 **위임**한다(중복 구현 안 함):
- **`premium-marketing`** — 앵커링·넛지·상욕방보 등 전환/가격 카피. `key-copy`가 활용.
- **`brand-story`** — 정체성/서사 앵글. `key-copy`가 활용.

> 없어도 이 플러그인 2개만으로 단독 동작한다. companion은 품질 부스터.

## 핵심 원칙
- **통과시키는 게 목적이 아니다.** item-check는 착수 전에 약한 곳을 드러낸다. 칭찬 말고 반박.
- **가상 수요 ≠ 실제 수요.** 인용하는 근거가 실제 인입/계약/인터뷰인지, 머릿속 예시인지 반드시 태그(✅/◐/❌).
- **한 개로 찍지 마라.** 키카피는 앵글이 다른 N개 변형으로 뽑아 시장이 고르게 한다.

## 설치
**A. 플러그인으로**: 이 폴더를 Claude Code 플러그인으로 설치.
**B. 스킬만 낱개로**:
```bash
cp -r skills/* ~/.claude/skills/
```
companion(`premium-marketing`·`brand-story`)을 함께 쓰려면 각자 설치.
